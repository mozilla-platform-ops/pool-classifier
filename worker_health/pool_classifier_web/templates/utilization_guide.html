Utilization API

Duration-weighted utilization is busy worker-seconds ÷ available worker-seconds × 100. It is not an average of bucket percentages.

curl '{{ request.host_url.rstrip('/') }}/api/v1/pools/{{ pool.provisioner }}/{{ pool.worker_type }}/utilization?start=2026-07-21T10:00:00Z&end=2026-07-21T12:00:00Z&bucket_seconds=3600'

Parameters and limits

start is inclusive and end exclusive; both are ISO 8601 timestamps with timezones. bucket_seconds is a positive integer. Ranges are limited to 90 days and 2,000 buckets.

Response and coverage

Each bucket returns busy_worker_hours and available_worker_hours (hours), worker_equivalents (average busy workers), utilization_pct, coverage_pct, complete, and status. Complete, zero-capacity buckets are unavailable; incomplete buckets have null metrics and report coverage. The summary endpoint anchors all standard windows to the latest common task and availability coverage boundary.

Availability and quarantine

recent_contact counts recently contacting, non-quarantined workers. listed counts non-quarantined workers returned by Taskcluster. Listed workers can be dormant or physically dead, so listing is not liveness confirmation. Quarantine excludes a worker from available capacity.

The repository's authoritative API documentation has the complete response schema and collection details.
